# Release Ticket — REQ-093

**Requirement:** Fix `compliance-evidence.yml` to upload `compliance/security-summary-<version>.md` for housekeeping releases
**Risk:** LOW
**Type:** fix (CI infrastructure)
**PR:** #496 (merged to develop 2026-07-14)

## Summary

`compliance/security-summary-<version>.md` files auto-generated by the housekeeping stub path were never uploaded to the DevAudit portal. The per-REQ evidence loop only scans `compliance/evidence/REQ-*/` and the release ticket loop only scans `compliance/pending-releases/`, so the root-level housekeeping security summary had no upload path. This caused the portal's `security_summary` gate to show MISSING for every housekeeping release, blocking release approval.

**Fix:** Added an explicit `shopt -s nullglob` glob for `compliance/security-summary-*.md` immediately before the release ticket upload block. Each file is uploaded as `security_summary` evidence against its version-derived release (version extracted from the filename).
