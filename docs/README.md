## 🔄 Keeping Documentation Updated

All documentation follows the standards defined in [DOCUMENTATION-STANDARD.md](./DOCUMENTATION-STANDARD.md).

**When to update documentation:**
- ✅ New features added
- ✅ API changes made
- ✅ Database schema modified
- ✅ Breaking changes introduced
- ✅ Configuration changes
- ✅ Complex bug fixes

**How to update documentation:**
1. Update relevant feature/technical documentation
2. Add entry to [CHANGELOG.md](./CHANGELOG.md)
3. Update this README if new categories added
4. Follow templates in [DOCUMENTATION-STANDARD.md](./DOCUMENTATION-STANDARD.md)
